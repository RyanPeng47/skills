# 协作接口指南

> 本指南帮助产品设计师与不同角色高效协作

## 与设计师协作

- **设计交接清单**: 确保原型包含所有状态（默认、悬停、禁用、加载、错误、空状态）
- **设计评审检查点**: 在原型阶段结束前进行设计评审
- **设计规范同步**: 确保设计系统/组件库的一致性

## 与开发协作

- **技术评审会议**: 在技术可行性阶段邀请开发参与
- **需求澄清机制**: 建立 RFC (Request for Comments) 流程
- **验收标准对齐**: 确保开发理解每个功能的验收条件
- **接口契约确认**: API 设计需双方确认

## 与QA协作

- **测试用例评审**: PRD 完成后与 QA 同步验收标准
- **边界条件说明**: 明确异常流程和边界情况
- **测试数据准备**: 提供测试场景和数据要求

## 与业务/运营协作

- **上线通知**: 提前同步发布计划
- **培训材料**: 准备用户指南或FAQ
- **反馈收集**: 建立用户反馈渠道
