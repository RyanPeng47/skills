# Agent 成本估算与优化指南
> 最后更新：2026-02-05（价格为示例范围，请按最新官方定价更新）

## 一、成本构成模型

Agent 产品的运行成本通常由以下几部分组成：

$$
总成本 = (输入Token费 + 输出Token费) + 向量存储/检索费 + 工具API调用费 + 基础设施费
$$

### 1.1 Token 消耗估算
-   **输入 (Input)**: System Prompt + 历史对话上下文 + RAG检索内容 + 用户当前输入
-   **输出 (Output)**: 模型生成的回答 +（如有）思考摘要/内部推理 + 工具调用指令
-   **倍率**: 中文通常 1 汉字 ≈ 0.6~0.8 Token (不同模型Tokenizer不同)

### 1.2 典型场景测算表 (示例：GPT-4o，仅供参考)

| 场景 | 单次输入 | 单次输出 | 交互轮数 | 单会话成本 (预估) | 1万次会话成本 |
|------|---------|---------|---------|------------------|--------------|
| **简单问答** | 500 toks | 200 toks | 1 | ~$0.005 | $50 |
| **文档分析** | 10,000 toks | 1,000 toks | 1 | ~$0.07 | $700 |
| **复杂Agent**| 2,000 toks | 500 toks | 10 | ~$0.20 | $2,000 |

> *注：Agent场景下，由于需携带System Prompt和History，且多轮交互，成本会通过轮数呈线性甚至指数级（如果Context累积）增长。*

## 二、成本优化策略 (Cost Optimization)

### 2.1 Prompt 优化
-   **精简 System Prompt**: 去除冗余描述。
-   **示例裁剪**: 动态加载 Few-shot 示例，只放入与当前Query最相关的示例，而不是全部。

### 2.2 流程优化
-   **模型分层 (Model Routing)**:
    -   意图识别/简单寒暄 -> 使用 **Flash/Mini** 模型 (便宜 10-50 倍)。
    -   复杂逻辑/代码生成 -> 使用 **Pro/GPT-4** 模型。
-   **缓存 (Caching)**:
    -   **语义缓存**: 同样的问题直接返回之前的答案。
    -   **Context Caching**: 对于超长文档，利用模型厂商提供的 Context Cache 功能（如 Claude Prompt Caching），大幅降低重复输入的费用。

### 2.3 RAG 优化
-   **控制 Context 长度**: 根据相关性分数截断检索结果，不要盲目塞入 Top-K。
-   **摘要索引**: 对长文档先做摘要，检索时先搜摘要，需要细节时再加载原文。

### 2.4 工程与监控
-   **Token 限制**: 在 API 调用层设置 `max_tokens` 防止模型发疯输出死循环。
-   **实时监控**: 建立 Token 消耗大盘，按 User/Tenant 维度监控异常流量。

## 三、ROI 分析框架

在立项阶段，应计算 ROI (投资回报率)：

$$
ROI = \frac{节省的人力成本 + 增加的业务价值 - Agent运行成本}{Agent开发成本 + Agent运行成本}
$$

-   **人力替代**: 这个任务原本人工做需要几分钟？人工时薪是多少？
-   **价值增益**: 24/7 服务带来的客户满意度提升、响应速度差异。

**决策红线**: 如果单次任务执行成本 > 人工执行成本的 50%，则需要重新评估技术方案或商业模式。
