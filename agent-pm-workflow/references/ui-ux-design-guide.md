# Agent UI/UX 设计指南

Agent 产品不同于传统 GUI 产品，核心在于 **"LUI (Language User Interface) + GUI"** 的混合交互。

## 一、交互模式选择

| 模式 | 描述 | 适用场景 | 优点 | 缺点 |
|------|------|---------|------|------|
| **Chat-Centric** | 标准对话框，类似 ChatGPT | 客服、咨询、通用助手 | 用户习惯、灵活性极高 | 信息展示效率低，操作繁琐 |
| **Copilot** | 侧边栏助手，伴随主应用 | 代码编辑、文档写作 | 不打断主流程，上下文感知强 | 屏幕占用，注意力分散 |
| **Agentic Workflow** | 画板/节点式流转 | 复杂工作流编排 | 过程透明，易于人工干预 | 交互门槛高 |
| **Invisible Agent** | 无对话界面，后台自动触发 | 邮件分类、自动填表 | 体验最流畅 (Magic) | 缺乏控制感，信任度建立难 |

## 二、Agent 特有界面元素

### 2.1 状态可视化 (State Visibility)
用户最怕"对着黑盒等待"。必须展示 Agent 在干什么：
-   **思考中 (Thinking)**: 展示 "正在规划任务..."，可折叠的思考摘要/步骤概览（避免暴露完整推理链）。
-   **工具调用 (Tool Use)**: "正在搜索 Google...", "正在读取文件...", "正在生成图表..."。
-   **输入中 (Typing)**: 流式输出 (Streaming) 是标配，打字机效果能缓解等待焦虑。

### 2.2 引用与溯源 (Citations)
建立信任的关键。
-   **RAG 场景**: 每句回答后必须通过 `[1]` 上标或卡片展示来源文档。点击可侧边栏展开原文并高亮。
-   **数据场景**: 图表下方标注数据来源 API 及时间。

### 2.3 人机协作控件 (Human-in-the-loop)
-   **快捷指令 (Chips)**: 在输入框上方提供 "下一步推荐"，减少用户输入。
-   **确认卡片**: 关键操作（如发送邮件、购买）前，弹出结构化卡片供用户 "Approve" 或 "Reject/Edit"。
-   **结果反馈**: 简单的 👍/👎 或 "重新生成"，用于收集 RLHF 数据。

## 三、对话体验设计原则

1.  **首选结构化输出，而非纯文本**:
    -   如果 Agent 生成列表，渲染为 Checkbox List。
    -   如果 Agent 生成数据，渲染为 Table 或 Chart。
    -   如果 Agent 生成代码，渲染为带有 Copy/Run 按钮的代码块。

2.  **错误处理与引导**:
    -   不要只说 "I don't know"。
    -   Agent 应该说 "我没有找到关于 X 的信息，但我可以帮你查一下 Y，或者你可以提供更多背景吗？"
    -   当意图不清时，主动发起 Clarification Question (澄清提问)。

3.  **个性化与语气**:
    -   根据 Agent 角色定义 UI 风格 (如律师 Agent 用深色严肃风，创意 Agent 用活泼风)。
    -   回复长度应适配终端（移动端简短，PC端详尽）。

## 四、设计检查清单

- [ ] 是否在首屏清楚说明了 Agent 能做什么，不能做什么？
- [ ] 等待时间是否有动态反馈？
- [ ] Markdown 渲染是否完整（表格、代码、公式）？
- [ ] 是否有方便的中止/重置 (Stop/Reset) 按钮？
- [ ] 历史记录是否易于回溯和管理？
